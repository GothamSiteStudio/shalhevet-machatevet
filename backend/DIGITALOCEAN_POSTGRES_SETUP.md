# חיבור PostgreSQL ב-DigitalOcean

אם בחרת לעבוד עם Neon במקום DigitalOcean, עברי למסמך `NEON_POSTGRES_SETUP.md`.

המטרה של המסמך הזה היא לקחת אותך צעד-צעד מהמצב הנוכחי של `db.json` למצב שבו הנתונים נשמרים במסד PostgreSQL אמיתי ב-DigitalOcean.

## מה אנחנו עושים עכשיו

בשלב הזה עדיין לא החלפנו את כל ה-backend לעבוד מול PostgreSQL בזמן ריצה.

כן הכנו 3 דברים חשובים:

1. בדיקת חיבור למסד: `npm run db:check`
2. יצירת הסכמה במסד: `npm run db:init:pg`
3. ייבוא הנתונים הקיימים מ-`db.json`: `npm run db:import:json`

ברגע שתסיימי את השלבים האלה, נעבור יחד לשלב הבא: החלפת שכבת הקריאה/כתיבה של השרת מ-JSON ל-PostgreSQL.

## לפני שמתחילים

צריך שיהיה לך:

1. חשבון פעיל ב-DigitalOcean
2. גישה לקובץ `backend/.env`
3. טרמינל פתוח בתיקיית הפרויקט

## שלב 1: יצירת PostgreSQL ב-DigitalOcean

1. התחברי ל-DigitalOcean.
2. בתפריט לחצי על `Create`.
3. בחרי `Databases`.
4. בחרי `PostgreSQL`.
5. לגרסת דמו ראשונה אפשר לבחור תוכנית קטנה וזולה.
6. בחרי אזור גאוגרפי קרוב אלייך.
7. תני למסד שם ברור, לדוגמה `shalhevet-db`.
8. לחצי על יצירה והמתיני עד שהמסד מוכן.

## שלב 2: פתיחת גישה מהמחשב שלך

DigitalOcean לא מאפשר חיבור מכל מקום כברירת מחדל. צריך לאשר את ה-IP שלך.

1. היכנסי למסך של מסד הנתונים שיצרת.
2. חפשי אזור בשם `Trusted Sources`.
3. הוסיפי את ה-IP הנוכחי של המחשב שלך.

אם לא תעשי את זה, בדיקת החיבור תיכשל גם אם כל הפרטים נכונים.

## שלב 3: העתקת פרטי החיבור

1. במסד הנתונים ב-DigitalOcean חפשי אזור בשם `Connection Details`.
2. העתיקי את ה-connection string המלא של PostgreSQL.
3. פתחי את הקובץ `backend/.env`.

הוסיפי או עדכני את השורות הבאות:

```env
FRONTEND_URL=http://localhost:3000
NGROK_URL=
DATABASE_URL=postgresql://USER:PASSWORD@HOST:25060/defaultdb?sslmode=require
PGSSL=true
```

הערות:

1. אל תמחקי את הערכים הקיימים של `JWT_SECRET` ופרטי המאמנת.
2. ב-DigitalOcean בדרך כלל `PGSSL=true` הוא הערך הנכון.

## שלב 4: בדיקת חיבור ראשונה

פתחי טרמינל בתיקיית `backend` והריצי:

```powershell
npm run db:check
```

אם הכל תקין, תראי הודעה שהחיבור ל-PostgreSQL הצליח.

אם זה נכשל:

1. בדקי שה-`DATABASE_URL` הודבק במלואו.
2. בדקי שה-IP שלך נוסף ל-`Trusted Sources`.
3. ודאי שלא הכנסת רווחים מיותרים ב-`.env`.

## שלב 5: יצירת הטבלאות במסד

אחרי שהחיבור הצליח, הריצי:

```powershell
npm run db:init:pg
```

הפקודה הזאת יוצרת את הטבלאות ומוודאת שיש חשבון מאמנת במסד.

## שלב 6: העברת הנתונים הקיימים

כדי לא לאבד את הנתונים שכבר קיימים ב-`db.json`, הריצי:

```powershell
npm run db:import:json
```

הפקודה הזו תעביר למסד:

1. משתמשים
2. היסטוריית משקל
3. עדכונים
4. פגישות
5. הודעות
6. יעדים אישיים
7. תפריטים אישיים
8. תוכניות אימון

## מה יקרה אחרי שתסיימי את 3 הפקודות

אחרי שתשלימי בהצלחה את:

1. `npm run db:check`
2. `npm run db:init:pg`
3. `npm run db:import:json`

נעבור לשלב הבא ונחליף את ה-backend עצמו לעבוד מול PostgreSQL במקום `db.json`.

זה השלב שבו הדמו כבר יהפוך להיות דמו על מסד אמיתי.

## מה לא לעשות כרגע

1. לא למחוק את `db.json` עדיין.
2. לא למחוק את `backend/.env`.
3. לא לפרוס עדיין ל-DigitalOcean App Platform לפני שבדקנו שהחיבור למסד עובד מהמחשב שלך.

## הסדר שאני ממליץ לך לעבוד בו

1. ליצור את מסד הנתונים ב-DigitalOcean
2. להוסיף Trusted Source
3. להדביק `DATABASE_URL` ב-`backend/.env`
4. להריץ `npm run db:check`
5. להריץ `npm run db:init:pg`
6. להריץ `npm run db:import:json`
7. לחזור אליי עם התוצאה של כל פקודה

אחרי זה אני אמשיך איתך לשלב הבא בצורה מסודרת.
