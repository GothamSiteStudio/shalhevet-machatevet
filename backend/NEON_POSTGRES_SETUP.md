# חיבור PostgreSQL ב-Neon

המטרה של המסמך הזה היא לקחת אותך צעד-צעד מהמצב הנוכחי של `db.json` למצב שבו הנתונים נשמרים במסד PostgreSQL אמיתי ב-Neon.

## מה אנחנו עושים עכשיו

בפרויקט הזה כבר השלמנו את המעבר של ה-backend ל-PostgreSQL בזמן ריצה.

שלוש הפקודות הבאות עדיין חשובות, כי הן מקימות את החיבור למסד חדש ומעבירות אליו את הנתונים הקיימים:

1. בדיקת חיבור למסד: `npm run db:check`
2. יצירת הסכמה במסד: `npm run db:init:pg`
3. ייבוא הנתונים הקיימים מ-`db.json`: `npm run db:import:json`

אחרי שהשלבים האלה מצליחים, השרת מוכן לעבוד מול Neon בפועל.

## לפני שמתחילים

צריך שיהיה לך:

1. חשבון פעיל ב-Neon
2. גישה לקובץ `backend/.env`
3. טרמינל פתוח בתיקיית הפרויקט

## שלב 1: יצירת פרויקט ב-Neon

1. היכנסי ל-`https://console.neon.tech`
2. התחברי או פתחי חשבון חדש
3. לחצי על `Create project`
4. תני לפרויקט שם ברור, לדוגמה `shalhevet-db`
5. בחרי אזור גאוגרפי קרוב ככל האפשר לשרת העתידי שלך
6. לחצי על יצירה והמתיני עד שהפרויקט מוכן

הערה חשובה:

1. Neon יוצר כברירת מחדל branch בשם `main`, מסד בשם `neondb`, ו-role ראשי. לדמו ראשון לא חייבים ליצור מסד חדש ידנית.

## שלב 2: העתקת פרטי החיבור

1. היכנסי למסך הפרויקט ב-Neon
2. לחצי על `Connect`
3. ודאי שה-branch הוא `main`
4. השאירי את ברירת המחדל של connection string כפי ש-Neon מציג
5. העתיקי את ה-connection string המלא שמתחיל ב-`postgresql://`

חשוב:

1. אל תמחוקי ידנית את הפרמטרים שבסוף הכתובת, למשל `sslmode=require` או `channel_binding=require`
2. ב-Neon לא צריך להוסיף `Trusted Sources` או לאשר IP מהמחשב שלך בשביל החיבור הבסיסי

## שלב 3: עדכון `backend/.env`

פתחי את הקובץ `backend/.env` והוסיפי או עדכני את השורות הבאות:

```env
FRONTEND_URL=
NGROK_URL=
DATABASE_URL=postgresql://USER:PASSWORD@HOST/DATABASE?sslmode=require&channel_binding=require
PGSSL=true
```

הערות:

1. את `DATABASE_URL` תדביקי בדיוק כפי שקיבלת מ-Neon
2. אל תמחקי את הערכים הקיימים של `JWT_SECRET` ופרטי המאמנת
3. `PGSSL=true` הוא הערך הנכון ל-Neon בפרויקט הזה

## שלב 4: בדיקת חיבור ראשונה

פתחי טרמינל בתיקיית `backend` והריצי:

```powershell
npm run db:check
```

אם הכל תקין, תראי הודעה שהחיבור ל-PostgreSQL הצליח.

אם זה נכשל:

1. בדקי שה-`DATABASE_URL` הודבק במלואו
2. ודאי שלא שינית ידנית חלקים מהכתובת שקיבלת מ-Neon
3. ודאי ש-`PGSSL=true`
4. אם Neon היה במצב idle, המתיני כמה שניות והריצי שוב

## שלב 5: יצירת הטבלאות במסד

אחרי שהחיבור הצליח, הריצי:

```powershell
npm run db:init:pg
```

הפקודה הזאת יוצרת את הטבלאות ומוודאת שיש חשבון מאמנת במסד.

## שלב 6: העברת הנתונים הקיימים

כדי לא לאבד את הנתונים שכבר קיימים ב-`db.json`, הריצי:

```powershell
npm run db:import:json
```

הפקודה הזו תעביר למסד:

1. משתמשים
2. היסטוריית משקל
3. עדכונים
4. פגישות
5. הודעות
6. יעדים אישיים
7. תפריטים אישיים
8. תוכניות אימון

## מה יקרה אחרי שתסיימי את 3 הפקודות

אחרי שתשלימי בהצלחה את:

1. `npm run db:check`
2. `npm run db:init:pg`
3. `npm run db:import:json`

ה-backend יוכל לעבוד מול PostgreSQL של Neon, והדמו שלך כבר ירוץ על מסד אמיתי.

בשלב הזה אפשר להפעיל את השרת והאפליקציה ולבדוק את הזרימה בפועל.

## מה לא לעשות כרגע

1. לא למחוק את `db.json` עדיין
2. לא למחוק את `backend/.env`
3. לא לשנות ידנית את מבנה ה-connection string של Neon

## הסדר שאני ממליץ לך לעבוד בו

1. ליצור פרויקט ב-Neon
2. להעתיק את `DATABASE_URL` מתוך `Connect`
3. להדביק אותו ב-`backend/.env`
4. להריץ `npm run db:check`
5. להריץ `npm run db:init:pg`
6. להריץ `npm run db:import:json`
7. לחזור אליי עם התוצאה של כל פקודה

אחרי זה אני אמשיך איתך לשלב הבא בצורה מסודרת.
