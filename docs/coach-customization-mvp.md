# MVP התאמות למאמנת ולמתאמנת

## מטרת ה-MVP
לבנות שכבת התאמה אישית שתיתן למאמנת שליטה עצמאית בתוך האפליקציה בלי צורך בכל פעם בפיתוח ידני.

ה-MVP מחולק לנושאים שניתן לשחרר בהדרגה ולבדוק מול שימוש אמיתי.

## נושאים שבוצעו
- [x] סטטוס ליווי ותגיות לקוחה
  - המאמנת יכולה לשמור סטטוס פנימי ותגיות לכל לקוחה.
  - הסטטוס והתגיות מוצגים בכרטיס הלקוחה בדשבורד.
  - נוספו פילטרים לפי סטטוס ותגיות בלוח הבקרה.

- [x] הרגלים ומשימות שהמאמנת מגדירה לבד
  - המאמנת יכולה להוסיף, לערוך, להשהות ולהפעיל הרגלים לכל לקוחה.
  - לכל הרגל יש שם, תדירות, יעד כמות והערה.
  - המתאמנת רואה את ההרגלים במסך המאמן ויכולה לסמן ביצוע.

- [x] צ'ק-אין שבועי דינמי
  - המאמנת יכולה לבנות טופס צ'ק-אין מותאם עם סוגי שאלות שונים.
  - המתאמנת ממלאת את הטופס מתוך האפליקציה.
  - ההגשה הראשונה של כל שבוע נשלחת גם לעדכונים של המאמנת.
  - במסך ניהול הלקוחה המאמנת רואה את הצ'ק-אין האחרון שהוגש.

## נושאים הבאים ב-MVP
- [ ] אוטומציות בסיסיות והתראות על לקוחות שלא מגיבות
- [ ] תבניות הודעות מהירות למאמנת
- [ ] תבניות תזונה ואימון לפי סוג לקוחה
- [ ] ספריית שאלוני פתיחה וצ'ק-אין לשכפול מהיר
- [ ] חלונות פגישה זמינים שהמאמנת מגדירה לבד
- [ ] ברודקאסט לקבוצות לקוחות לפי תגיות
- [ ] דגלים אדומים ולקוחות שדורשות טיפול
- [ ] דוחות היענות שבועיים למאמנת

## מה לבדוק עכשיו
1. להיכנס כמאמנת, לפתוח לקוחה, ולהגדיר סטטוס ליווי ותגיות.
2. לוודא שהסטטוס והתגיות מופיעים בכרטיס הלקוחה ושאפשר לסנן לפיהם בדשבורד.
3. להיכנס ללקוחה, להוסיף 2-3 הרגלים, ולשמור.
4. להיכנס כמתאמנת ולוודא שההרגלים מופיעים במסך המאמן ואפשר לסמן אותם.
5. להגדיר טופס צ'ק-אין שבועי עם כמה סוגי שאלות.
6. להיכנס כמתאמנת, למלא את הצ'ק-אין, ולוודא שהוא נשמר ומופיע אצל המאמנת.

## הערות מימוש
- הנתונים נשמרים ב-PostgreSQL.
- ההרגלים ותבנית הצ'ק-אין נשמרים כחלק מפרופיל הלקוחה.
- הגשות צ'ק-אין והשלמות הרגלים נשמרות בנפרד כדי לאפשר התרחבות בהמשך.
