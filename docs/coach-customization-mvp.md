# MVP התאמות למאמנת ולמתאמנת

## מטרת ה-MVP
לבנות שכבת התאמה אישית שתיתן למאמנת שליטה עצמאית בתוך האפליקציה בלי צורך בכל פעם בפיתוח ידני.

ה-MVP מחולק לנושאים שניתן לשחרר בהדרגה ולבדוק מול שימוש אמיתי.

## נושאים שבוצעו
- [x] סטטוס ליווי ותגיות לקוחה
  - המאמנת יכולה לשמור סטטוס פנימי ותגיות לכל לקוחה.
  - הסטטוס והתגיות מוצגים בכרטיס הלקוחה בדשבורד.
  - נוספו פילטרים לפי סטטוס ותגיות בלוח הבקרה.

- [x] הרגלים ומשימות שהמאמנת מגדירה לבד
  - המאמנת יכולה להוסיף, לערוך, להשהות ולהפעיל הרגלים לכל לקוחה.
  - לכל הרגל יש שם, תדירות, יעד כמות והערה.
  - המתאמנת רואה את ההרגלים במסך המאמן ויכולה לסמן ביצוע.

- [x] צ'ק-אין שבועי דינמי
  - המאמנת יכולה לבנות טופס צ'ק-אין מותאם עם סוגי שאלות שונים.
  - המתאמנת ממלאת את הטופס מתוך האפליקציה.
  - ההגשה הראשונה של כל שבוע נשלחת גם לעדכונים של המאמנת.
  - במסך ניהול הלקוחה המאמנת רואה את הצ'ק-אין האחרון שהוגש.

- [x] אוטומציות בסיסיות והתראות על לקוחות שלא מגיבות
  - הדשבורד מזהה לקוחות שלא הראו פעילות בזמן ומציג דגלים, סיכום וסינון ייעודי.
  - במסך הלקוחה המאמנת רואה למה הלקוחה סומנה: חוסר פעילות, צ׳ק-אין חסר, הרגלים שלא סומנו או חוסר תגובה אחרי הודעת מעקב.
  - נוספה תזכורת אוטומטית מוכנה מראש שאפשר לשלוח ישירות מתוך מסך הלקוחה, עם cooldown כדי לא להציף הודעות.

- [x] תבניות הודעות מהירות למאמנת
  - המאמנת יכולה לשמור סט גלובלי של תבניות הודעה מהירות לשימוש חוזר מול כל הלקוחות.
  - מתוך מסך הלקוחה אפשר לערוך, למחוק, להוסיף ולשמור תבניות בלי לצאת מהחשבון של הלקוחה.
  - לכל תבנית יש תצוגה מקדימה עם שם הלקוחה בפועל ואפשר לשלוח אותה בלחיצה אחת.

## נושאים הבאים ב-MVP
- [ ] תבניות תזונה ואימון לפי סוג לקוחה
- [ ] ספריית שאלוני פתיחה וצ'ק-אין לשכפול מהיר
- [ ] חלונות פגישה זמינים שהמאמנת מגדירה לבד
- [ ] ברודקאסט לקבוצות לקוחות לפי תגיות
- [ ] דגלים אדומים ולקוחות שדורשות טיפול
- [ ] דוחות היענות שבועיים למאמנת

## מה לבדוק עכשיו
1. להיכנס כמאמנת, לפתוח לקוחה כלשהי ולעבור לטאב החשבון. לוודא שמופיע אזור חדש של "הודעות מהירות".
2. לבדוק שמופיעות תבניות ברירת מחדל, לערוך לפחות תבנית אחת, להוסיף תבנית חדשה ולשמור. לסגור ולפתוח שוב לקוחה כדי לוודא שהתבניות נשמרו.
3. בתוך אחת התבניות, להשתמש ב-{{firstName}} או ב-{{name}} ולוודא שבתצוגה המקדימה מופיע השם האמיתי של הלקוחה.
4. לשלוח תבנית ללקוחה ולוודא שמתקבלת הודעת הצלחה ושסטטוס "תזכורת אחרונה" באזור המעקב האוטומטי מתעדכן.
5. לחזור לדשבורד, לרענן, ולוודא שהלקוחה משקפת פעילות אחרונה של הודעת מאמנת אם זה משפיע על סטטוס המעקב.

## הערות מימוש
- הנתונים נשמרים ב-PostgreSQL.
- ההרגלים ותבנית הצ'ק-אין נשמרים כחלק מפרופיל הלקוחה.
- תבניות הודעה מהירה נשמרות ברמת חשבון המאמנת ומשותפות לכל הלקוחות.
- הגשות צ'ק-אין והשלמות הרגלים נשמרות בנפרד כדי לאפשר התרחבות בהמשך.
- התראות חוסר תגובה מחושבות לפי פעילות לקוחה בפועל: עדכונים, הודעות מצד הלקוחה, בקשות פגישה, צ׳ק-אין, סימוני הרגלים ושקילות.
- שליחת תזכורת אוטומטית נעשית כהודעת מאמנת רגילה עם נוסח מוצע לפי הסיבה לדגל, ומוגבלת אוטומטית לחלון cooldown של 24 שעות.
- שליחה מהירה מתוך תבנית משתמשת באותו ערוץ הודעות של המאמנת, כך שהיא משפיעה על חישוב המעקב האוטומטי כמו כל הודעת מאמנת אחרת.
